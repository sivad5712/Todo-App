                    STEP 3: INSTALLING GIT
===============================================================================

Git is a tool that helps you save and share your code. You need it to upload
your project to GitHub.

WHAT IS GIT?
Git is like a "save game" system for code. It tracks all changes you make
and helps you share your work with others.

---------------------------------------------
FOR WINDOWS:
---------------------------------------------

1. Go to: https://git-scm.com/download/windows
2. The download should start automatically
3. Run the downloaded file (Git-2.xx.x-64-bit.exe)
4. Installation wizard will appear:
   
   - Click "Next" on the welcome screen
   - Keep the default installation location, click "Next"
   - Keep all default options selected, click "Next"
   - Choose default editor (Vim is fine), click "Next"
   - Choose "Git from the command line and also from 3rd-party software", click "Next"
   - Choose "Use bundled OpenSSH", click "Next"
   - Choose "Use the OpenSSL library", click "Next"
   - Choose "Checkout Windows-style, commit Unix-style line endings", click "Next"
   - Choose "Use MinTTY", click "Next"
   - Keep default options for remaining screens, click "Next"
   - Click "Install"
   - Click "Finish" when done

5. Verify installation:
   - Press Windows key + R
   - Type: cmd
   - Press Enter
   - Type: git --version
   - Press Enter
   - You should see something like: git version 2.43.0

‚úì Git is now installed!

---------------------------------------------
FOR MAC:
---------------------------------------------

Method 1 - Using Homebrew (easier if you have it):
1. Open Terminal (search for "Terminal" in Spotlight)
2. Type: brew install git
3. Press Enter

Method 2 - Direct Download:
1. Go to: https://git-scm.com/download/mac
2. Download the installer
3. Open the downloaded .dmg file
4. Follow the installation instructions
5. Open Terminal and type: git --version
6. You should see the Git version number

Method 3 - Using Xcode Command Line Tools:
1. Open Terminal
2. Type: xcode-select --install
3. Press Enter
4. Follow the prompts to install
5. Type: git --version to verify

‚úì Git is now installed!

---------------------------------------------
FOR LINUX (Ubuntu/Debian):
---------------------------------------------

1. Open Terminal (Ctrl + Alt + T)
2. Type: sudo apt-get update
3. Press Enter (enter your password if asked)
4. Type: sudo apt-get install git
5. Press Enter
6. Type: y when asked to confirm
7. Type: git --version
8. You should see the Git version number

For other Linux distributions:
- Fedora: sudo dnf install git
- Arch: sudo pacman -S git
- OpenSUSE: sudo zypper install git

‚úì Git is now installed!

===============================================================================
                    OPTION B: MANUAL COMMANDS
===============================================================================

If you want to learn Git commands, or if the script doesn't work, use this method.

üí° These instructions work on Windows (Command Prompt/PowerShell), Mac (Terminal), 
   and Linux (Terminal).

---------------------------------------------
1. OPEN YOUR TERMINAL/COMMAND PROMPT
---------------------------------------------

Windows:
  - Press Windows key + R
  - Type: cmd
  - Press Enter

Mac:
  - Press Cmd + Space
  - Type: Terminal
  - Press Enter

Linux:
  - Press Ctrl + Alt + T

---------------------------------------------
2. NAVIGATE TO YOUR PROJECT FOLDER
---------------------------------------------

Type: cd 
(that's "cd" followed by a space)

Then type or drag the path to your todo_assessment folder

Example for Windows:
  cd C:\Users\YourName\Desktop\todo_assessment

Example for Mac/Linux:
  cd /Users/YourName/Desktop/todo_assessment

Press Enter

üí° TIP: You can drag and drop the folder into the terminal to auto-fill the path!

---------------------------------------------
3. CONFIGURE GIT (FIRST TIME ONLY)
---------------------------------------------

Tell Git who you are:

Type: git config --global user.name "Your Full Name"
Example: git config --global user.name "John Smith"
Press Enter

Type: git config --global user.email "your.email@example.com"
Example: git config --global user.email "john.smith@email.com"
üí° Use the same email as your GitHub account!
Press Enter

---------------------------------------------
4. INITIALIZE GIT REPOSITORY
---------------------------------------------

Type: git init
Press Enter

You should see: "Initialized empty Git repository..."

---------------------------------------------
5. ADD ALL FILES
---------------------------------------------

Type: git add .
(that's "git add" followed by a space and a period)
Press Enter

This tells Git to track all files in your project.

---------------------------------------------
6. CREATE YOUR FIRST COMMIT
---------------------------------------------

Type: git commit -m "Initial commit: ECHO Todo Assessment submission"
Press Enter

You should see a list of files that were committed.

WHAT IS A COMMIT?
A commit is like taking a snapshot of your project at this moment.

---------------------------------------------
7. RENAME BRANCH TO MAIN
---------------------------------------------

Type: git branch -M main
Press Enter

This creates/renames your branch to "main" (GitHub's default).

---------------------------------------------
8. CONNECT TO YOUR GITHUB REPOSITORY
---------------------------------------------

Type: git remote add origin YOUR_REPOSITORY_URL
Replace YOUR_REPOSITORY_URL with the URL you copied from GitHub!

Example: 
  git remote add origin https://github.com/johnsmith/echo-todo-assessment.git

Press Enter

---------------------------------------------
9. PUSH TO GITHUB
---------------------------------------------

Type: git push -u origin main
Press Enter

You'll be asked for credentials:
  - Username: Your GitHub username
  - Password: Your Personal Access Token (NOT your GitHub password!)

DON'T HAVE A TOKEN? See "Creating a Personal Access Token" below!

TROUBLESHOOTING:
If you get an error like "failed to push", try:
  git push -u origin main --force

Wait for the upload to complete...

You should see: "Branch 'main' set up to track remote branch 'main' from 'origin'."

‚úì Done! Continue to Step 8.

===============================================================================
            CREATING A PERSONAL ACCESS TOKEN (PAT)
===============================================================================

GitHub no longer accepts passwords when pushing code. You need a "Personal 
Access Token" instead. Think of it as a special password just for code uploads.

TO CREATE YOUR TOKEN:

1. Log into GitHub: https://github.com
2. Click your profile picture (top right corner)
3. Click "Settings"
4. Scroll down and click "Developer settings" (at the bottom of the left menu)
5. Click "Personal access tokens"
6. Click "Tokens (classic)"
7. Click "Generate new token"
8. Click "Generate new token (classic)"
9. GitHub might ask for your password - enter it
10. Fill in the form:

    Note: 
    - Type: "ECHO Todo Assessment Upload"
    
    Expiration:
    - Choose: 30 days (or longer if you want)
    
    Scopes (permissions):
    - Check the box: "repo" (this will check all repo sub-boxes)
    - That's all you need!

11. Scroll down and click "Generate token"
12. YOU'LL SEE A LONG STRING OF LETTERS AND NUMBERS!
    
    ‚ö†Ô∏è IMPORTANT: Copy this token NOW!
    GitHub will NEVER show it again!
    
    Click the copy button next to the token.

13. SAVE THIS TOKEN SOMEWHERE SAFE!
    - Paste it in a text file
    - Or write it down
    
    üíæ Your token: _______________________________________________

14. Use this token as your "password" when Git asks for credentials!

===============================================================================
              STEP 10: TROUBLESHOOTING COMMON PROBLEMS
===============================================================================

---------------------------------------------
PROBLEM: "git is not recognized as a command"
---------------------------------------------

SOLUTION:
- Git is not installed or not in your system PATH
- Reinstall Git (see Step 3)
- On Windows: Restart your computer after installing Git
- Try closing and reopening your terminal/command prompt

---------------------------------------------
PROBLEM: "Permission denied (publickey)"
---------------------------------------------

SOLUTION:
- You're using SSH instead of HTTPS
- Make sure your repository URL starts with "https://" not "git@"
- Use: https://github.com/username/repo.git

---------------------------------------------
PROBLEM: "Authentication failed"
---------------------------------------------

SOLUTION:
- You're using your GitHub password instead of a Personal Access Token
- Create a Personal Access Token (see instructions above)
- Use the token as your password when Git asks

---------------------------------------------
PROBLEM: "Repository not found"
---------------------------------------------

SOLUTION:
- Check that you typed the repository URL correctly
- Make sure the repository exists on GitHub
- Verify you're logged into the correct GitHub account
- The repository might be private (make it public in settings)

---------------------------------------------
PROBLEM: "Failed to push - rejected"
---------------------------------------------

SOLUTION:
- The repository already has files
- Try: git push -u origin main --force
- ‚ö†Ô∏è Warning: This will overwrite the remote repository!
- Only do this if you're sure

---------------------------------------------
PROBLEM: Script won't run on Mac/Linux
---------------------------------------------

SOLUTION:
- Make the script executable first
- Type: chmod +x push_to_github.sh
- Then run: ./push_to_github.sh

---------------------------------------------
PROBLEM: "Nothing to commit"
---------------------------------------------

SOLUTION:
- All files might already be committed
- Check with: git status
- If you made changes, use: git add .
- Then: git commit -m "Your message"
- Then: git push

---------------------------------------------
PROBLEM: Can't find the extracted folder
---------------------------------------------

SOLUTION:
- Search your computer for "todo_assessment"
- Check your Downloads folder
- Check your Desktop
- Re-extract the ZIP file if needed

---------------------------------------------
PROBLEM: Node.js installation fails
---------------------------------------------

SOLUTION:
- Make sure you downloaded the correct version for your OS
- Try downloading the installer again
- On Mac: Try using Homebrew: brew install node
- On Linux: Try using your package manager

---------------------------------------------
PROBLEM: ZIP file won't extract
---------------------------------------------

SOLUTION:
- The download might be corrupted
- Download the ZIP file again
- On Windows: Try using 7-Zip (free software)
- On Mac: Try using The Unarchiver (free app)

---------------------------------------------
PROBLEM: Script hangs or freezes
---------------------------------------------

SOLUTION:
- Press Ctrl+C to cancel
- Close the terminal and try again
- Try the manual method instead
- Check your internet connection

===============================================================================
                              GLOSSARY
===============================================================================

Git: A version control system that tracks changes in your code
GitHub: A website where you can store and share your Git repositories
Repository (Repo): A folder that contains your project and its history
Commit: A snapshot of your project at a specific point in time
Push: Uploading your commits to GitHub
Clone: Downloading a repository from GitHub to your computer
Branch: A separate version of your code (like a parallel universe)
Main/Master: The primary branch of your repository
Remote: A version of your repository stored on GitHub
Origin: The default name for your remote repository
Terminal/Command Prompt: A text-based interface for running commands
CLI (Command Line Interface): Another term for Terminal/Command Prompt
Personal Access Token (PAT): A special password for Git operations
HTTPS: A secure way to connect to GitHub (vs SSH)
